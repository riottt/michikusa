# MICHIKUSA プロダクト仕様

## 1. プロダクト定義

MICHIKUSAは、行きたい場所が思いついていない人へ、現在地から始まる外出を一つ決めるAI外出エージェントです。

```text
現在地と空き時間を取得
        ↓
AIが行動可能な範囲を決める
        ↓
行き先と順番を決める
        ↓
各地点での過ごし方を決める
        ↓
帰る時刻までタイムライン化
        ↓
地図とGoogleカレンダーへ反映
        ↓
状況変化に応じて残りを再計画
```

MICHIKUSAが作るのは店舗一覧ではありません。「移動する」「見つける」「その場で少し過ごす」「無理なく終える」をつないだ道草セッションです。

## 2. 解決する状態

### 家にいる人

- 休日だが予定がない。
- 外には出たいが、店や場所を調べる気力がない。
- 候補を比較しているうちに昼を過ぎる。
- 行き先が決まらず、画面を眺めたまま一日が終わる。

この場合は、徒歩だけに閉じず、電車や自転車を含む60〜180分の外出を作ります。最初の行動は5〜12分程度に抑え、家を出る負担を小さくします。

### 外にいる人

- 用事が終わったが、まだ帰るには早い。
- 次に行く場所がなく、同じ店へ戻りそうになる。
- 近くを検索して比較するほどの気力はない。

この場合は、徒歩を中心に20〜90分の寄り道を作ります。帰宅方向や次の予定を崩さない範囲で、2〜3地点をつなぎます。

## 3. ユーザーへの約束

1. 候補一覧を見せず、一つのルートを決める。
2. 最初に所要時間、予算、帰宅目安だけを保証する。
3. その時点で必要な次の行動だけを大きく見せる。
4. 場所だけでなく、そこで何をして楽しむかまで決める。
5. 遅延、休業、疲労、帰宅希望があれば残りだけを組み直す。
6. Calendarへの書き込みは、人の承認後に行う。
7. 共有するのは正確な座標ではなく、体験の型と記憶である。

## 4. 体験状態

画面はページを増やさず、一枚の地図が状態に応じて変わります。

| 状態 | 地図上の変化 | 主操作 |
|---|---|---|
| 待機 | 現在地だけを表示 | 今日を動かす |
| 計画中 | 候補点、採否、ピン、ルートを順に描画 | エージェントの動きを見る |
| 完成 | 2〜4地点と時間割を表示 | この道草で出発 |
| 移動 | 現在のピンだけを強調 | 着いた |
| 現地 | 写真、短いタイマー、完了操作 | 見つけた／できた |
| 完了 | 抽象化した経路とLUCKをカード化 | シェア／保存 |

## 5. 道草セッションの構造

| 役割 | 目的 | 例 |
|---|---|---|
| DISCOVER | 普段と違う場所へ向かう | 路地、書店、展示、神社、知らない出口 |
| STAY | 少し立ち止まる | 公園、図書室、喫茶店、川沿い |
| LANDING | 無理なく終える | 帰宅方向の休憩地点、駅、明るい店 |

現地の遊びは一地点につき一つです。

```text
表紙だけで気になる本を選び、10分だけ読む。
一番気になった色を写真に残す。
水辺で好きな曲を一曲だけ聴く。
普段選ばない飲み物を一つ頼む。
```

## 6. エージェントの責務

- 家／外の状態を判定する。
- Calendarから空き時間と次の予定を確認する。
- 移動手段と探索半径を決める。
- Placesから営業中の候補を集める。
- 過去の訪問と重複しすぎないようにする。
- 移動時間、予算、場所の種類を比較する。
- 2〜4地点の順番を組む。
- 各地点の短い遊びを生成する。
- 営業時間、予算、夜間、帰宅余白を検査する。
- Calendarへ登録可能な移動・滞在イベントを作る。
- 遅延や休業時に残りだけを組み直す。
- 完了後の呼び名と共有カードを作る。

## 7. LUCK

LUCKは生産性や連続利用を競う点数ではなく、偶然の発見を残す軽い記録です。

- ランキングは作らない。
- 連続利用が途切れても減点しない。
- 一地点あたり10〜25程度。
- 完了した地点分だけ加算する。
- 後から生活感のある呼び名を付ける。

例:

- 夕方から動き出す者
- 予定なき沿線旅人
- 路地裏を信じる者
- 無目的散歩師

## 8. 共有

共有カードへ含めるもの:

- エリア名
- 抽象化したルート
- 地点数、距離、時間
- 獲得LUCK
- その日の呼び名
- ユーザーが選んだ写真

含めないもの:

- 自宅の正確な座標
- Calendarの予定タイトル
- OAuth情報
- 全移動履歴

将来の共有導線は「この道草をあなたの街で再生成する」です。同じ店を共有するのではなく、「路地を歩く→書店で一冊選ぶ→公園で読む」のような体験の型を、閲覧者の現在地で作り直します。

## 9. 作らない機能

- TODO・タスク管理
- 店舗ランキング
- レビュー比較
- 長いAIチャット
- フォロー、コメント、常時位置共有
- 複雑なレベル育成
- 全地点での証拠提出
- 無断のCalendar書き込み

## 10. 成功を測る指標

中心指標:

> ルート生成後30分以内に最初のピンへ到着した割合

補助指標:

- 「今日を動かす」の押下率
- 生成から出発までの時間
- 最初のピン到着率
- 全地点完了率
- 再計画率と理由
- Calendar登録率
- 共有率
- 7日以内の再利用率
