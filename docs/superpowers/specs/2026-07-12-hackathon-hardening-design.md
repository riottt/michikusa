# MICHIKUSA ハッカソン提出強化設計

## 目的

MICHIKUSAの既存の一画面体験と世界観を維持したまま、審査員が90秒で「本当に動くAgentであり、失敗時にも説明できる」と判断できる状態へ引き上げる。

## 採用方針

機能追加型、演出追加型、技術実在性優先型の3案を比較し、技術実在性優先型を採用する。新しいSNS、ランキング、チャット、複雑なゲーム要素は追加しない。既存の開始、計画、出発、現地行動、再計画、完了の流れに、実行モード、Calendar状態、再計画の結果、安全確認の意味を補う。

## 変更する体験

### 実行モードの明示

計画結果に含まれる `source` を、提案カード上で `LIVE`、`DEMO`、`FALLBACK` として表示する。デモデータを実データに見せない。ヘルスAPIはAgentの `demo_mode`、Maps/Geminiの接続状態、ADKバージョンを安全な公開情報として返し、本番スモークテストで実連携を判定できるようにする。

### Calendarの正確な案内

Calendar未接続時のデモcommitは、Calendarへ登録したような文言を出さない。「道草を開始しました。カレンダーは未接続です」と表示する。実接続時だけ「カレンダーに予定を追加しました」と表示する。

### 再計画結果の可視化

再計画の理由、地点数の増減、所要時間の増減、新しい帰宅目安をクライアントで比較し、再計画後の移動カードに短い結果として表示する。完了後まで保持する必要はなく、次の再計画または新しい計画開始で置き換える。

### 安全確認とアクセシビリティ

数値だけの安全スコアを `安全確認 98` のように意味のある表示へ変える。再計画メニューボタンへ明示的なアクセシブル名を付ける。既存の淡い配色とレイアウトは維持し、審査デモに必要な情報だけを追加する。

## Agent評価と提出証拠

既存のADKグラフテストに、予算、帰宅時刻、営業時間、共有情報、再計画理由ごとの契約を追加する。E2Eは実行モード、Calendar未接続文言、再計画結果、アクセシブル名を検査する。`docs/VERIFICATION.md` は実際のコマンドと件数へ更新し、本番デプロイ後に公開URL、health、plan、replanの結果を記録する。

## データフロー

1. WebがAgentのhealthを取得し、安全な接続状態を `/api/health` で返す。
2. Agentのplan `source` をUIが提案カードへ表示する。
3. 出発時にCalendar commit結果の `demo` を見て正確な文言を表示する。
4. 再計画前後のplanを比較してクライアント側でreceiptを作り、次の行動と一緒に表示する。
5. 本番スモークテストはhealthのlive flagsとplanの `source` を検査する。

## エラー処理

- Agent healthが取得できない場合は既存どおり503を返し、接続情報を推測しない。
- Calendar commitが409または失敗した場合もルート開始は継続する。
- 再計画が失敗した場合は元のplanを保持する。
- 本番デプロイは必要なSecretが揃っていない場合に開始前検査で停止する。

## テスト戦略

- RED: E2EとAgent契約テストへ新しい期待値を追加し、現行実装で失敗を確認する。
- GREEN: UI、health、Agent guardテストを最小変更で通す。
- REFACTOR: 再計画比較を純粋関数へ分離し、表示コンポーネントから計算を外す。
- VERIFY: `npm run verify`、`npm run test:e2e`、`npm audit --omit=dev --audit-level=high`、secret scan、Cloud Runスモークを実行する。

## デプロイ設計

既存の `deploy.sh` を使い、専用Google Cloudプロジェクト、Secret Manager、Artifact Registry、非公開Agent Cloud Run、公開Web Cloud Runを構成する。WebだけがAgentを呼び出せるIAMを維持し、両サービスはmin instances 0、max instances 1を既定にする。デプロイ後は公開URLでhealth、ホーム、plan、Calendarの未接続挙動、replanを確認する。

## 完了条件

- 審査上の誤解を生む4つのUI問題が自動テストで防止される。
- Agent契約テストが予算、帰宅、安全、共有情報を検査する。
- 全ローカル検証が通る。
- PR経由で保護されたmainへ反映される。
- Cloud Runの公開Webが200を返し、Agent healthがlive接続状態を返す。
